── 6. TemplateRenderer — Protocol Review Workflow ───────────────────────────
Full workflow implementation:

LINEAR LIFECYCLE (enforced):
  draft → in_review → approved → published
  needs_changes available from in_review or approved → returns to in_review
  published is terminal (no further transitions)
  reset always available (admin escape hatch)

TRANSITION GATING:
  Buttons enabled/disabled based on current state.
  Disabled buttons show tooltip explaining required state.
  Visual pipeline strip shows current position + completed steps (✓).

CONFIRMATION MODALS (all transitions require confirmation):
  Mark In Review  — simple confirm, no notes field
  Needs Changes   — reason field (recommended), destructive red button
  Approve         — optional notes field
  Publish         — optional notes field, high-stakes warning in body text
  Reset           — separate destructive confirm, "cannot be undone" warning
